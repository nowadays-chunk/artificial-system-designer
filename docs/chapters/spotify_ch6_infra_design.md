# Spotify Case Study - Chapter 6: Security Architecture

## 1. Digital Rights Management (DRM) and Playback Control

To protect intellectual property and comply with licensing agreements, we secure the audio streaming pipeline:

```
       [ Client Request + Signed Token ] ---> [ CDN Edge ]
                                                 |
                                +----------------+----------------+
                                | (Valid Token)                   | (Invalid Token)
                                v                                 v
                       [ Stream Encrypted Ogg ]          [ HTTP 403 Forbidden ]
                                |
                                v
                       [ Decrypt via KMS Key ]
```

- **Encrypted Media Assets**: Audio files are stored and streamed in encrypted format.
- **Signed Tokens**: To access the CDN stream, the client must present a signed token generated by the Playback Session Service.
- **Key Decryption**: The client player fetches content decryption keys from a Key Management Service (KMS) over a secure channel. Key generation is rate-limited to prevent automated content ripping.

---

## 2. API Gateway Security Integration

- **OAuth 2.0 Access Scopes**: Access to user playlists and profile settings is controlled using OAuth 2.0 access scopes (e.g., `playlist-read-private`, `user-library-modify`).
- **Internal Authentication**: Inter-service communication is secured using mutual TLS (mTLS) to prevent man-in-the-middle attacks within the service mesh.
- **WAF Protections**: Ingress gateways block SQL injection, cross-site scripting (XSS), and automated scraping traffic.
